I'm considering how to best use Antigravity, the new agentic tool from Google.

My solution is to create a single repository for managing and developing my projects.

It contains a structured lifecycle for transforming raw concepts into managed, scaling projects. This permits me to develop multiple projects in parallel by freely switching between them and is designed such that, if I make changes to my development process, these changes may be mirrored across all projects at once.

I refer to this system as a "garden" and consider it to be an effective use of Antigravity as AI tends to be "high-volume but low signal" and "process-oriented rather than outcome-oriented" which are characteristics that favor a parallel, allometric approach.
